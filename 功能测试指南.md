# 朗读功能测试指南

## ⚠️ 当前方案说明

### 已知限制

**问题**: 微信小程序对音频播放有严格限制
- 外部TTS API(有道、百度)返回的音频无法直接播放
- `Unable to decode audio data` 错误
- 浏览器SpeechSynthesis仅在某些环境可用

**当前方案**:
- 主要使用浏览器SpeechSynthesis
- 仅在开发者工具等特定环境可用
- **真机上可能不可用**

### 建议方案

如需在真机上使用语音朗读,建议:
1. **接入微信同声传译插件** (推荐)
2. **使用腾讯云TTS + 云函数代理**
3. **后端服务器处理TTS,返回音频**

## 问题修复说明

已修复 `tencentTtsService is not defined` 错误和音频解码问题。

## 测试步骤

### 1. 测试原文朗读

1. 打开任意作业详情页
2. 确保"朗读模式"为"原文"
3. 点击"点我读作业"大按钮
4. 预期结果:
   - 显示"正在朗读原文"提示
   - 开始播放中文语音
   - 按钮变为红色"点我停止"

### 2. 测试翻译朗读

1. 打开任意作业详情页
2. 点击"翻译"按钮切换到翻译模式
3. 选择目标语言(英语)
4. 点击"读翻译内容"按钮
5. 预期结果:
   - 先显示"准备朗读..."
   - 显示"正在朗读译文(英语)"
   - 播放英语语音

### 3. 测试语言切换

1. 在翻译模式下,点击"中文"按钮
2. 再次点击"读翻译内容"
3. 预期结果:
   - 显示"正在朗读译文(中文)"
   - 播放中文语音

### 4. 测试停止朗读

1. 在播放过程中点击"点我停止"
2. 预期结果:
   - 音频立即停止
   - 按钮恢复绿色
   - `isReading` 状态变为 false

## 常见问题排查

### 问题1: 点击后无响应

**可能原因**:
- `tencentTtsService` 未正确导入
- 文件路径错误

**检查方法**:
```javascript
// 检查 detail.js 第1-5行
const tencentTtsService = require('../../utils/tencent-tts-service');
```

### 问题2: 音频解码失败 (Unable to decode audio data)

**可能原因**:
- TTS API返回的音频格式不支持
- HTTP协议被小程序限制
- 网络问题导致音频损坏

**解决方案**:
- 已切换到百度TTS(支持HTTPS)
- 会自动尝试浏览器TTS备选方案
- 检查网络连接

### 问题3: 显示"请求超时"

**可能原因**:
- 有道翻译API限流
- 文本过长

**解决方案**:
- 失败时会返回原文本
- 建议分段处理长文本
