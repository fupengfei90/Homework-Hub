# 微信群消息同步测试指南

## 📋 前提条件检查

在开始测试之前，请确认以下条件已满足：

- [x] 已开通云开发
- [x] 已配置环境ID
- [ ] 已部署 `syncMessage` 云函数
- [ ] 已创建数据库集合 `sync_messages`
- [ ] 已配置HTTP触发器

## 🚀 完整测试步骤

### 第1步：部署syncMessage云函数（5分钟）

在微信开发者工具中：

1. 找到 `cloudfunctions/syncMessage/` 文件夹
2. 右键点击 → "上传并部署：云端安装依赖"
3. 等待部署完成（约1-2分钟）

### 第2步：创建数据库集合（2分钟）

在云开发控制台中：

1. 打开云开发控制台
2. 点击左侧菜单"数据库"
3. 点击"添加集合"
4. 集合名称：`sync_messages`
5. 权限：选择"仅创建者可读写"
6. 点击"确定"

### 第3步：添加HTTP触发器（2分钟）

在云开发控制台中：

1. 点击左侧菜单"云函数"
2. 点击 `syncMessage` 云函数
3. 点击"触发器"标签
4. 点击"添加触发器"
5. 选择"HTTP触发器"
6. 路径设置为：`/syncMessage`
7. 点击"确定"
8. **复制触发器URL**（格式：`https://xxx-xxx.service.tcloudbase.com/syncMessage`）

### 第4步：测试HTTP触发器（1分钟）

使用 curl 或 Postman 测试触发器是否正常：

**方法1：使用 curl**
```bash
curl -X POST https://你的触发器URL/syncMessage \
  -H "Content-Type: application/json" \
  -d '{
    "action": "saveMessage",
    "message": {
      "content": "这是一条测试消息",
      "senderName": "测试老师",
      "groupName": "测试群",
      "messageType": "notice"
    }
  }'
```

**方法2：使用 Python 测试脚本**
```bash
cd itchat-sync
python test_cloud_function.py
```

预期返回结果：
```json
{
  "code": 0,
  "message": "操作成功"
}
```

### 第5步：验证数据库（1分钟）

在云开发控制台中：

1. 点击左侧菜单"数据库"
2. 点击 `sync_messages` 集合
3. 查看是否有刚刚测试的记录
4. 应该能看到一条类似这样的记录：
```json
{
  "_id": "xxx",
  "content": "这是一条测试消息",
  "senderName": "测试老师",
  "groupName": "测试群",
  "messageType": "notice",
  "createTime": 1234567890123,
  "formatTime": "今天 14:30"
}
```

# 修改后（替换成你的真实URL）

## 🔧 常见问题排查

### 问题1：云函数部署失败

**解决方案：**
- 检查环境ID是否正确
- 检查网络连接
- 尝试重新部署

### 问题2：HTTP触发器请求失败

**解决方案：**
- 检查触发器URL是否正确
- 检查防火墙是否允许访问
- 查看云函数日志

### 问题3：itchat扫码失败

**解决方案：**
- 检查itchat版本是否最新
- 尝试重新安装依赖
- 检查网络连接

### 问题4：消息未同步到数据库

**解决方案：**
- 检查HTTP_API_URL配置
- 查看itchat终端输出
- 查看云函数日志

### 问题5：小程序未显示消息

**解决方案：**
- 下拉刷新页面
- 检查数据库是否有数据
- 查看小程序控制台错误信息
